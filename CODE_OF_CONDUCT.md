# Code of Conduct

Racna is a small open-source project. We want contributors, users, and
maintainers to be able to take part without friction. This document spells
out what's expected of anyone interacting in project spaces (issues, pull
requests, discussions, and any other space where the project is represented).

## The short version

Be respectful. Assume good intent. Critique ideas, not people. Accept
constructive feedback gracefully, and offer it the same way.

## What we encourage

- **Welcoming language** and a friendly tone, especially toward newcomers
- **Constructive feedback** focused on the work, not the person
- **Patience** with contributors who are still learning
- **Honesty** about mistakes, including your own
- **Crediting** others' contributions and prior work

## What is not acceptable

- Personal attacks, insults, or demeaning remarks
- Discriminatory remarks of any kind
- Unwanted personal contact after being asked to stop
- Publishing someone's private information without their consent
- Sustained disruption of discussions
- Any conduct that a reasonable person would consider inappropriate in a
  professional setting

## Scope

This applies in every space where the project is represented: GitHub issues
and pull requests, discussions, project chat (if any), and public spaces when
acting on the project's behalf.

## Reporting

If you see behavior that violates this code, or you are on the receiving end
of it, please report it privately to **racna.dev@gmail.com**.

Reports will be:

- Acknowledged within 48 hours
- Reviewed confidentially
- Acted on proportionately (a quiet word, removal of comments, temporary or
  permanent ban from project spaces)

Maintainers who act on reports will recuse themselves if they are personally
involved.

## Standard

This Code of Conduct is adapted from the
[Contributor Covenant, version 2.1](https://www.contributor-covenant.org/version/2/1/code_of_conduct/),
which is the binding reference if any wording here is ambiguous.
